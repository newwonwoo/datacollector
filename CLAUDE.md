# CLAUDE.md

## 작업 시작 시 필수 읽기

이 리포에서 작업하기 전에 아래 파일을 **반드시 먼저 읽는다**:

1. **`docs/GOTCHAS.md`** — 과거 세션에서 발생한 실수의 영구 목록. 같은 실수 반복 금지.
2. **`Master_01~03.md`** — 설계서 본문 (v10).
3. **`docs/Appendix_A~D.md`** — 설계서 부록.

## 작업 규칙

### 우선순위 (상위 순)
1. 사용자 요청 (최우선)
2. 기능중단급 오류 (P0)
3. 주요 기능 일부 오류 (P1)
4. 비주요 기능 오류 (P2)
5. 개선·보완 (P3)
6. 신규 기능 (P4)

### 병렬 실행
각 작업 시작 전 병렬 가능 여부를 검토한다. 독립 파일·함수 변경은 한 메시지에서 parallel Write/Edit 로 묶는다.

### 하네스 (재작업 최소화)
각 작업마다:
- **DoD**: 설계서 몇 절 충족 + 테스트 ID 명시.
- **Invariant**: 기존 테스트 regression 없을 것.
- **Blast Radius**: 건드리는 파일 명시.

커밋 전 `python -m pytest` 필수 통과.

### 사용자 환경 확인
답변 전 확인: 모바일/PC, 터미널 가능?, 리포 Public/Private?, 어디까지 설정 완료?

## 보안
- `.env`에만 실제 키. `.env.example`는 플레이스홀더 전용.
- 채팅/로그/커밋 메시지에 토큰·API키·비밀번호 **절대** 에코하지 않는다.
- 유저가 채팅에 비밀값 붙여넣으면 즉시 revoke 안내.

## 테스트 실행
```bash
python -m pytest
```
- 189+ 테스트 전원 통과 + 1초 내 완료가 기본선.
- 실패 시 원인 파악 후 커밋 금지.
